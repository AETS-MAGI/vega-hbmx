### 発見した新しいエビデンス（ソース逐次確認）

#### 1. CMake デフォルトフィルタの正確な regex（ollama/CMakeLists.txt:125-128）

```cmake
if(NOT AMDGPU_TARGETS)
    find_package(hip REQUIRED)
    list(FILTER AMDGPU_TARGETS INCLUDE REGEX "^gfx(94[012]|101[02]|1030|110[012]|120[01])$")
endif()
```

- 許可されるターゲット：gfx940/941/942（CDNA3）、gfx1010/1012（RDNA1）、gfx1030（RDNA2）、gfx1100/1101/1102（RDNA3）、gfx1200/1201（RDNA4）
- **gfx900 はこの regex に含まれない**。デフォルトビルドでは HIP ターゲットとして選択されない
- 明示的に `AMDGPU_TARGETS=gfx900` を環境変数でセットすれば `if(NOT AMDGPU_TARGETS)` ブロックをスキップしてビルド可能
- 手元の `/usr/lib/ollama/libggml-hip.so` はこの override 経由か事前コンパイル済みアーティファクトと判断

#### 2. HIP バックエンドにおける gfx900 のアーキテクチャ定数と dp4a 実装（ggml-cuda/common.cuh）

```cpp
// Line 65
#define GGML_CUDA_CC_VEGA     (GGML_CUDA_CC_OFFSET_AMD + 0x900)  // Vega56/64, minimum for fp16 dual issue
// Line 66
#define GGML_CUDA_CC_VEGA20   (GGML_CUDA_CC_OFFSET_AMD + 0x906)  // MI50/Radeon VII, minimum for dp4a
```

gfx900 は `GGML_CUDA_CC_VEGA`（0x1000900）として定義されており、dp4a のハードウェア最低ラインである `GGML_CUDA_CC_VEGA20`（0x1000906）の手前にいる。

`ggml_cuda_dp4a()` 関数（lines 666-704）の実装分岐：

| 条件 | 実装 | コスト |
|---|---|---|
| CDNA / RDNA2 / `__gfx906__` | `__builtin_amdgcn_sdot4(a, b, c, false)` | 1命令（ハードウェア） |
| RDNA3 / RDNA4 | `__builtin_amdgcn_sudot4(...)` | 1命令（ハードウェア） |
| **RDNA1 / `__gfx900__`** | **6命令インラインアセンブリ（v_mul_i32_i24 ×4 + v_add3_u32 ×2）** | **ソフトウェアエミュレーション** |

また、`V_DOT2_F32_F16_AVAILABLE`（line 715）のマクロ定義からも gfx900 は除外されており、fp16 のドット積命令（`v_dot2_f32_f16`）は利用不可。

これらは gfx900 が HIP バックエンドで「動く」が演算単位では最適化の恩恵を受けにくい構造であることを示す。

#### 3. Vulkan バックエンドにおける gfx900 の AMD_GCN 分類とその影響（ggml-vulkan/ggml-vulkan.cpp）

アーキテクチャ判定ロジック（lines 261-297）：

```cpp
if (subgroup_size_control_props.maxSubgroupSize == 64 &&
    subgroup_size_control_props.minSubgroupSize == 64) {
    return vk_device_architecture::AMD_GCN;  // line 297
}
```

Vega56（gfx900）は wave64 固定のため、この条件に合致して `AMD_GCN` に分類される。

`AMD_GCN` に対する機能制限（複数箇所で確認）：

```cpp
// line 3964
const bool use_subgroups =
    device->subgroup_arithmetic && device->architecture != vk_device_architecture::AMD_GCN;
```

→ gfx900 では subgroup arithmetic 最適化が **明示的に無効化**されている。

```cpp
// lines 3241-3243: AMD_GCN + open-source driver 向け warptile
if ((device->architecture == AMD_GCN) && (device->driver_id != vk::DriverId::eAmdProprietary)) {
    m_warptile_mmq = m_warptile_mmq_int = { 256, 64, 64, 32, 16, 16, 2, 2, 2, 1, 16 };
}
```

→ 行列積の tile パラメータも RDNA 系とは異なるセットが使われる。

ジャーナル実測値（01:31:37 JST）：

```
ggml_vulkan: 0 = AMD Radeon RX Vega (RADV VEGA10) (radv) |
  uma: 0 | fp16: 1 | bf16: 0 | warp size: 64 |
  shared memory: 65536 | int dot: 0 | matrix cores: none
```

- `int dot: 0`：VK_KHR_shader_integer_dot_product がハードウェアアクセラレートされていない
- `matrix cores: none`：cooperative matrix サポートなし
- これらはソースコードの分岐条件と整合している
