## Linux Kernel Architecture Overview

The Linux kernel is the core system software that manages hardware resources, provides basic system services, and runs all applications on the machine. Its architecture integrates several subsystems working in concert to ensure stability, performance, and extensibility. Below is an explanation of key components:

---

### 1. **Scheduler (Process Scheduling)**

The kernel's job scheduling algorithm manages CPU time among multiple threads or processes to achieve efficient task execution.

**Key Concepts:**
- **Preemptive Scheduling**: Processes can be preempted from running to allow other tasks to use CPU time.
- **Cron Job Scheduler**: Runs periodic jobs in background at specific intervals.
- **Load Balancing (NUMA)**: Distributes workloads across memory-intensive tasks for better performance.
- **Process Priority Levels**: Assign different priorities to prioritize high-priority processes.

**Scheduling Modes:**
1. **Round Robin**: Processes are given a time slice in order; each gets CPU once per cycle.
2. **Priority Queues**: Lower priority processes may be preempts based on their assigned levels.
3. **Completely Fair Scheduler (CFS)**: Allocates resources fairly across all runnable tasks using `cfs` scheduler's fairness metric.
4. **Throttling/Preemption**: Used for controlling interrupt handling and CPU load balancing.

---

### 2. **Memory Management**

- **Memory Types**:
  - **Physical Memory (RAM)**: Accessed only by the kernel for data structures and page tables.
