The Linux kernel is built around five core subsystems that coordinate system operation: the scheduler, memory management, virtual file system (VFS), interrupts, and device drivers. These subsystems work together to manage system resources and ensure efficient, deterministic execution.

---

### 1. Overall Architecture

The Linux kernel is modular and organized into subsystems, each encapsulated and managed as a distinct entity:

- **Core Module Structure**:
  - The kernel is primarily divided into modules such as kernel headers, subsystems (e.g., networking, drivers), drivers, and the system-level components (e.g., scheduler).
- **Boot and Bootloader Support**:
  - The kernel initializes during boot and can be split across memory regions (e.g., RAM or ROM).
  - A boot loader (such as GRUB, U-Boot, or similar) loads and sets up the kernel.
- **Inter-process and Inter-subsystem Communication**:
  - The kernel supports multiple concurrent processes, each executing its own code.
  - System-level interactions occur between processes, subsystems, and hardware.
